# vds-icon

The `vds-icon` web component renders glyphs from the shared VDS icon font so icons stay consistent across frameworks.

## Installation

```bash
npm install @vds/icon
```

## Usage

```html
<script type="module" src="@vds/web-components/icon"></script>

<vds-icon name="check" label="Success"></vds-icon>
<vds-icon name="close" style="font-size:2rem" color="var(--vds-color-error)"></vds-icon>
```

### Properties

| Property     | Type                     | Default | Description                                    |
|--------------|--------------------------|---------|------------------------------------------------|
| `name`       | `string`                 | `''`    | Icon name, mapped to the IcoMoon CSS class (`v-{name}`) |
| `color`      | `string`                 | `null`  | Custom color (any valid CSS color)             |
| `label`      | `string`                 | `null`  | Accessible label. Required unless decorative   |
| `decorative` | `boolean`                | `false` | Marks icon as decorative (`aria-hidden="true"`) |
| `spin`       | `boolean`                | `false` | Adds a continuous spin animation               |
| `rotate`     | `number`                 | `0`     | Rotates the icon by the provided degrees       |

> **Sizing**: The icon inherits `font-size` from wherever you place it. Use utility classes or inline styles (e.g., `.text-lg` or `style="font-size:24px"`) to scale the icon just like text.

## Using the IcoMoon CSS

The component is a lightweight wrapper around the CSS classes generated by IcoMoon. To ensure the glyphs render:

1. Export your icon set from IcoMoon and copy the output `style.css` and `fonts/` directory into `packages/vds-icon/assets/`.
2. Load the generated CSS once in your host application (for example, import it from a global stylesheet or include it in your bundler entry point).
3. Use the base icon names (`meeting`, `download`, etc.). The component automatically applies the prefixed class (`v-meeting`) expected by the CSS.

Because the glyphs come directly from the CSS, you don’t need to regenerate any TypeScript map—just keep the CSS and font files in sync with your IcoMoon export.

## Accessibility

- Provide a `label` whenever the icon conveys meaning (e.g., `"Success"`).
- Use the `decorative` attribute for purely visual icons so they’re hidden from assistive tech.
